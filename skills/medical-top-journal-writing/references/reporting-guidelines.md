## Fast mapping

Map the study design to the nearest reporting standard before drafting.

| Study design or output | Default reporting standard | Practical drafting note |
| --- | --- | --- |
| Randomized clinical trial | CONSORT | Include allocation, flow, registration, protocol, primary outcome, harms |
| Cluster, noninferiority, pragmatic, nonpharmacologic RCT variants | CONSORT plus relevant extension | Mention the extension if the variant changes methods or flow |
| Nonrandomized intervention study | TREND | Make assignment logic and intervention delivery explicit |
| Cohort, case-control, cross-sectional study | STROBE | Clarify population, exposure, outcome, missingness, and bias |
| Systematic review or meta-analysis | PRISMA | Make search, eligibility, study selection, and evidence synthesis auditable |
| Diagnostic accuracy study | STARD | Report index test, reference standard, thresholds, and accuracy measures |
| Prediction model development or validation | TRIPOD | Separate development, validation, calibration, and discrimination |
| Quality improvement study | SQUIRE | Anchor the intervention to the care setting and implementation context |
| Economic evaluation | CHEERS | Show perspective, time horizon, costing, and uncertainty analysis |
| Qualitative study | SRQR or COREQ | Explain sampling, reflexivity, analytic process, and trustworthiness |
| Case report | CARE | Build a clean chronology and clinical rationale |
| Genetic association study | STREGA | Clarify variant definitions, population handling, and multiple testing |
| Survey study | AAPOR best practices, plus STROBE logic where relevant | Report sampling frame, response rate, and nonresponse bias handling |

## Design-specific reminders

### CONSORT-family work

- Keep the primary outcome definition identical across protocol, abstract, results, and tables.
- Report allocation, masking, participant flow, harms, and deviations from protocol.
- Registration ID and protocol availability should never appear as an afterthought.

### STROBE-family work

- Use association language unless the causal framework is explicit and defensible.
- State inclusion and exclusion rules, exposure timing, confounding strategy, missing data handling, and sensitivity analyses.
- Report the study dates and setting cleanly.

### PRISMA-family work

- Make the search reproducible enough for editorial review.
- Quantify records identified, screened, excluded, and included.
- State whether synthesis is narrative, quantitative, or both.

### STARD and TRIPOD work

- Separate model derivation from validation.
- Specify thresholds, reference standards, calibration, discrimination, and clinical utility if available.
- Avoid vague claims such as "high accuracy" without the actual performance metrics.

### SQUIRE, SRQR, COREQ, and CARE work

- Explain context, process, and interpretation rather than only listing findings.
- For qualitative research, show who was interviewed, how themes were derived, and how credibility was strengthened.
- For case reports, maintain a transparent chronology and clearly justified diagnostic reasoning.

## Submission-side artifacts

Expect to prepare some or all of these alongside the manuscript:

- checklist for the reporting standard
- participant flow diagram or study selection flow
- protocol or SAP
- registry record or ID
- supplemental methods or survey instrument
- evidence-quality table for reviews when the journal expects it

If the user does not know the exact reporting guideline, infer the best match, state the assumption, and proceed.
